# AI Open Problems Hackathon

Статический прототип сайта для летнего хакатона исследовательских задач.

## Что уже есть

- карточки задач с фильтрами по статусу, направлению и году;
- страница задачи с паспортом, условием, скрытыми блоками обзора, прогресса, комментариев и использования ИИ;
- форма отклика по каждой задаче без регистрации на сайте;
- форма предложения новой задачи;
- предзаполненные ссылки на GitHub issue, письмо и копирование текста.

## Как запустить

Откройте `index.html` в браузере. Сервер не нужен.

## Проверка базы

Перед публикацией запустите:

```bash
npm run validate
```

Проверка ловит битую кириллицу, незакрытые TeX-разделители, сырые математические записи вне TeX, непереведенные латинские фрагменты, битые ссылки и пропущенные поля. Правила качества карточек описаны в `RULES.md`.

## Где менять данные

Стартовая база лежит в `TASKS` внутри `app.js`.

Перед публикацией нужно заменить `CONFIG.contactEmail` на рабочий адрес организатора, если почтовая обратная связь остается включенной.

## Дальнейшая архитектура

Следующий естественный шаг - вынести `TASKS` в отдельный JSON или подключить GitHub Issues как источник данных. Для полноценной обратной связи без регистрации можно добавить внешний form endpoint или небольшой serverless endpoint, который будет принимать предложения и создавать issue от имени бота.
