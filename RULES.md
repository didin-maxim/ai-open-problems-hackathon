# Правила базы задач

База должна быть полезна студенту-решателю без дополнительного контекста.

## Карточка задачи

- Видимая часть карточки самодостаточна: понятны объекты, параметры и вопрос.
- В паспорте отдельно указаны автор математической постановки и тот, кто предложил задачу в базу.
- Статус выбирается из фиксированного списка: открытая проблема; есть неопубликованное решение; есть малоизвестное опубликованное решение; решено с помощью ИИ и пока не опубликовано.
- Год постановки, первоисточник и обзор литературы указываются настолько точно, насколько это возможно. Сомнения записываются явно.
- При открытии страницы сразу видны только условие и паспорт. Подсказки, обзор, прогресс, комментарии, ИИ и обратная связь раскрываются по клику.

## Текст и формулы

- Весь пояснительный текст пишется по-русски.
- Английские термины оставляются только как названия статей, стратегий, моделей или поисковые ключи.
- Все формулы набираются в TeX через `\(...\)`.
- Сырые записи вроде `n >= 3`, `K_{3,n}` или `lim s(n)/n` не должны попадать в обычный текст задачи.
- Не допускаются битая кириллица, символ `�`, mojibake-фрагменты вроде `Рџ` или `СЃ`, а также незакрытые TeX-разделители.

## ИИ

- Если задача предложена или найдена ИИ, это отмечается в поле «Предложил» и в метках.
- При решении можно пользоваться любыми ИИ.
- В отчете нужно указать модель, роль ИИ, этап работы и что было проверено человеком.

## Публикация

Перед публикацией нужно запустить:

```bash
npm run validate
```

Валидатор проверяет обязательные поля, статусы, ссылки, битую кириллицу, TeX-разделители, подозрительные непереведенные куски и сырые математические записи.
